The `@zayne-labs/eslint-config` package exports individual config functions that can be used directly if you prefer more granular control over your ESLint configuration.

## Core Configs

### javascript()

Core JavaScript ESLint configuration with recommended rules.

```typescript
import { javascript } from "@zayne-labs/eslint-config";

export default [
  ...await javascript({
    overrides: {
      "no-console": "warn",
    },
  }),
];
```

<ParamField path="options" type="object">
  <Expandable title="Properties">
    <ParamField path="overrides" type="Record<string, RuleEntry>">
      Override specific JavaScript rule configurations.
    </ParamField>
  </Expandable>
</ParamField>

**Key Features:**
- ECMAScript latest with module support
- Browser and Node.js globals
- Strict error detection rules
- Enforces modern JavaScript patterns
- Complexity limits (max depth: 2, complexity: 50)

**Notable Rules:**
- `no-console`: Error (allows warn, error, info, trace)
- `prefer-const`: Error with destructuring support
- `no-var`: Error
- `eqeqeq`: Error (always, except for null)
- `curly`: Error (multi-line)

### typescript()

TypeScript ESLint configuration with optional type-aware rules.

```typescript
import { typescript } from "@zayne-labs/eslint-config";

export default [
  ...await typescript({
    tsconfigPath: "./tsconfig.json",
    isTypeAware: true,
    stylistic: true,
  }),
];
```

<ParamField path="options" type="object">
  <Expandable title="Properties">
    <ParamField path="tsconfigPath" type="true | string | string[] | null" default="true">
      Path to tsconfig.json for type-aware rules. Set to `true` for auto-discovery.
    </ParamField>

    <ParamField path="isTypeAware" type="boolean" default="false">
      Enable type-aware rules. Automatically `true` if `tsconfigPath` is provided.
    </ParamField>

    <ParamField path="files" type="string[]" default="['**/*.ts', '**/*.tsx', ...componentExts]">
      Glob patterns for TypeScript files.
    </ParamField>

    <ParamField path="filesTypeAware" type="string[]" default="['**/*.ts', '**/*.tsx', ...componentExtsTypeAware]">
      Glob patterns for type-aware files.
    </ParamField>

    <ParamField path="ignoresTypeAware" type="string[]" default="['**/*.md/**', '**/*.astro/*.ts']">
      Patterns to exclude from type checking.
    </ParamField>

    <ParamField path="componentExts" type="string[]" default="[]">
      Additional component file extensions (e.g., `['vue']`).
    </ParamField>

    <ParamField path="componentExtsTypeAware" type="string[]" default="[]">
      Component extensions that should be type-aware.
    </ParamField>

    <ParamField path="erasableOnly" type="boolean" default="false">
      Enable erasable syntax only rules from `eslint-plugin-erasable-syntax-only`.
    </ParamField>

    <ParamField path="stylistic" type="boolean" default="true">
      Enable TypeScript stylistic rules.
    </ParamField>

    <ParamField path="allowDefaultProject" type="string[]" default="undefined">
      Default projects to allow in the parser project service (max 8).
    </ParamField>

    <ParamField path="parserOptions" type="ParserOptions">
      Additional TypeScript parser options.
    </ParamField>

    <ParamField path="overrides" type="Record<string, RuleEntry>">
      Override specific TypeScript rule configurations.
    </ParamField>
  </Expandable>
</ParamField>

**Key Features:**
- Strict or strict-type-checked presets
- Type-aware rules when tsconfig provided
- Stylistic TypeScript rules
- Component extension support

**Notable Rules:**
- `ts-eslint/consistent-type-definitions`: Type aliases preferred
- `ts-eslint/array-type`: Array-simple style
- `ts-eslint/no-unused-vars`: Warning with underscore prefix support
- `ts-eslint/prefer-nullish-coalescing`: Error (ignores conditional tests)
- `ts-eslint/no-import-type-side-effects`: Error

## Framework Configs

### react()

Comprehensive React linting with hooks, compiler, and refresh support.

```typescript
import { react } from "@zayne-labs/eslint-config";

export default [
  ...await react({
    typescript: true,
    compiler: true,
    refresh: true,
    nextjs: true,
  }),
];
```

<ParamField path="options" type="object">
  <Expandable title="Properties">
    <ParamField path="react" type="boolean | object" default="true">
      Enable default React rules from `@eslint-react/eslint-plugin`.

      When passing an object:
      - `overrides`: Override specific React rule configurations
    </ParamField>

    <ParamField path="compiler" type="boolean | object" default="true">
      Enable React compiler rules for optimization.

      When passing an object:
      - `overrides`: Override specific compiler rule configurations
    </ParamField>

    <ParamField path="refresh" type="boolean | object" default="true">
      Enable react-refresh/HMR rules.

      When passing an object:
      - `overrides`: Override specific refresh rule configurations
    </ParamField>

    <ParamField path="youMightNotNeedAnEffect" type="boolean | object" default="true">
      Enable rules to avoid unnecessary useEffect usage.

      When passing an object:
      - `overrides`: Override specific rule configurations
    </ParamField>

    <ParamField path="nextjs" type="boolean | object" default="auto-detect">
      Enable Next.js specific rules. Auto-detects if Next.js is installed.

      When passing an object:
      - `overrides`: Override specific Next.js rule configurations
    </ParamField>

    <ParamField path="typescript" type="boolean" default="true">
      Enable TypeScript support for React files.
    </ParamField>

    <ParamField path="files" type="string[]" default="['src/**/*']">
      Glob patterns for React files.
    </ParamField>

    <ParamField path="filesTypeAware" type="string[]" default="['**/*.ts', '**/*.tsx']">
      Patterns for type-aware React files.
    </ParamField>

    <ParamField path="ignoresTypeAware" type="string[]" default="['**/*.md/**', '**/*.astro/*.ts']">
      Patterns to exclude from type checking.
    </ParamField>

    <ParamField path="overrides" type="Record<string, RuleEntry>">
      Override specific React rule configurations.
    </ParamField>
  </Expandable>
</ParamField>

**Required Plugins:**
- `@eslint-react/eslint-plugin`
- `eslint-plugin-react-hooks`
- `eslint-plugin-react-refresh`
- `eslint-plugin-react-you-might-not-need-an-effect`
- `@next/eslint-plugin-next` (if Next.js is used)

## Usage Example

Combine individual configs for full control:

```javascript title="eslint.config.js"
import {
  javascript,
  typescript,
  react,
  stylistic,
  imports,
  jsonc,
} from "@zayne-labs/eslint-config";

export default [
  ...await javascript(),
  ...await typescript({
    tsconfigPath: "./tsconfig.json",
    isTypeAware: true,
  }),
  ...await react({
    nextjs: true,
    compiler: true,
  }),
  ...await stylistic({
    indent: 2,
    quotes: "single",
    jsx: true,
  }),
  ...await imports({
    typescript: true,
    stylistic: true,
  }),
  ...await jsonc(),
  {
    files: ["**/*.tsx"],
    rules: {
      "react-x/no-clone-element": "error",
    },
  },
];
```
